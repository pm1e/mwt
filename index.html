Цели Аналоги

    Планирование маршрутов – важная задача транспортной логистики, в ходе решения которой следует учитывать множество параметров:

  • пожелания клиентов по времени доставки;
  • дорожные пробки;
  • габариты, масса товара и вместимость транспортного средства.

    Автоматизация данного процесса позволяет:

  • снизить временные затраты на планирование маршрутов и исключить возможные ошибки при ручном планировании;
  • сократить временные и материальные затраты на транспортировку.

Данный ресурс посвящен вопросам проектирования и реализации системы для автоматизации процесса планирования маршрутов. Задачи, связанные с данным процессом, пути и алгоритмы их решения также будут проанализированы в рамках работы.

Цели

Требуется разработать программный продукт для поддержки и оптимизации логистической деятельности организации, который включает в себя:

  • web-интерфейс для логиста;
  • web-интерфейс для администратора;
  • мобильное приложение для исполнителей.

    Логисты управляют складскими запасами и заказами, занимаются организацией и координацией грузоперевозок, составляют маршруты грузоперевозок и координируют работу водителей. Web-интерфейс для данных специалистов должен предоставлять следующие возможности:

  • работа с заявками на доставку, т.е. добавление, просмотр, изменение и удаление заявок;
  • выбор исполнителей, ручное или автоматическое распределение заявок между ними;
  • автоматическое составление маршрута для каждого исполнителя и возможность его ручной корректировки;
  • контроль за процессом доставки

Web-интерфейс администратора должен предоставлять возможность работы с данными пользователей: логистов, исполнителей, клиентов, а также с информацией о транспортных средствах

    Мобильное приложение исполнителя должно решать следующие задачи:

  • отображение маршрута до пунктов назначения на карте;
  • отображение контактной информации о соответствующих клиентах;
  • изменение статуса заявки доставки.

Интерфейс взаимодействия с пользователем должен выглядеть легко, быть удобным и интуитивно понятным. Web-интерфейсы и интерфейс мобильного приложения должны корректно отображаться на экранах с различным разрешением.

Аналоги

    Существует ряд многофункциональных платформ для решения логистических задач, наиболее популярными из них являются следующие:

  • Maxoptra
  • ЯКурьер
  • Ant logistics
  • Delans
  • Яндекс.Маршрутизация

    Данные платформы имеют схожий функционал и предоставляют следующие возможности:

  • настройка зон доставки
  • интеграции с онлайн кассами позволяет фиксировать денежные поступления и осуществлять возвраты по заказам
  • формирование различных отчетов
  • обеспечивают коммуникацию с исполнителем и клиентами
  • ведение учета по складу

    Все системы позволяют осуществлять распределение заказов между исполнителями и планирование оптимальных маршрутов с учетом множества требований:

  • время погрузки и разгрузки
  • дорожные пробки
  • скоростные режимы
  • тип дороги

Платформы также имеют и схожие интерфейсы.

    Рассмотрим основные элементы интерфейса на примере сервиса Ant logistics:

  • верхняя панель приложения содержит различные элементы управления и строку поиска пункта доставки
  • панель инструментов включает в себя средства для работы с картой и часто используемые команды
  • карта с нанесенными на нее точками, складами и маршрутами располагается в центральной части интерфейса
  • справочники служат для хранения информации о точках, машинах и других данных, используемых для составления маршрутов.
  • документы содержат список заявок на доставку, построенные маршруты, план-факт анализ движения водителей по маршрутам.

Интерфейс платформы Maxoptra более удобный, т.к. он не перегружен большим количеством элементов, все элементы распределены по категориям, навигация по которым осуществляется с помощью верхней панели.

Муравьиный алгоритм Генетический алгоритм

Разрабатываемая система решает задачу маршрутизации транспортных средств (VRP). Данная задача заключается в поиске маршрутов для парка транспортных средств, посещающих несколько точек.

Частным случаем VRP является задача коммивояжера, целью которой является поиск самого выгодного маршрута, проходящего через набор удаленных друг от друга точек один раз, с дальнейшим возвратом в начальную точку.

    VRP может иметь дополнительные ограничения, в связи с этим можно выделить следующие виды данной задачи:

  • Capacitated VRP (CPRV) – задача с ограничением вместимости транспортных средств, т.е. Суммарный объем товаров, доставляемых клиентам, не должен превышать общую вместимость транспортных средств.
  • Multiple Depot VRP (MDVRP) – случай, в котором клиенты обслуживаются с нескольких складов (депо), каждый клиент закрепляется за определенным депо.
  • Periodic VRP (PVRP) – задача, в которой для каждого клиента задана частота посещения в течение заданного периода времени.
  • Split Delivery VRP (SDVRP) – вариант задачи, в котором один клиент может обслуживаться разными курьерами. Это возможно, например, при разделении заказов на более маленькие.
  • VRP with Pick-Up and Delivering (VRPPD) – задача, в которой клиенты могут возвращать товары. В данном случае возможны дополнительные временные затраты на перестановку грузов, если заказы доставлены еще не всем потребителям.
  • VRP with Backhauls (VRPB) – аналог VRPPD с дополнительным ограничением, которое заключается в том, что возвраты товаров могут осуществляться только после окончания всех доставок.
  • VRP with Satellite Facilities (VRPSF) – случай, в котором в процессе доставки возможна дозагрузка транспортных средств.
  • VRP with Time Windows (VRPTW) – задача с временными окнами, т.е. заказ должен быть доставлен в заранее определенный промежуток времени.

    Существует несколько подходов к решению данных, рассмотрим их на примере задачи коммивояжера.

  • точные методы позволяют найти оптимальное решение путем вычисления всех возможных вариантов. Поиск точного решения может потребовать ощутимых временных затрат, поэтому на практике часто достаточно найти приближенное решение.
  • Эвристические методы позволяют найти допустимое решение за хорошее время благодаря тому, что пространство поиска решений ограничивается.
  • Метаэвристические методы позволяют получать более качественные решения, чем классические эвристики. Они более гибкие и позволяют учитывать множество ограничений. В данных алгоритмах упор делается на глубокое исследование многообещающих областей пространства решений.

В разрабатываемом программном продукте будут использоваться именно метаэвристические методы, т.к. они позволяют быстро получать хорошее решение даже при достаточно большой размерности задачи. В основе практически всех метаэвристик лежат идеи, появившиеся при наблюдении за природой. Далее будут подробно рассмотрены два наиболее популярных алгоритма: муравьиный и генетический.

Муравьиный алгоритм (англ. – ant colony optimization, ACO) моделирует поведение муравьиной колонии. В природе муравьи беспорядочно блуждают в поисках пищи, найдя ее, они возвращаются в свою колонию, прокладывая феромонные следы. При нахождении данного пути другие муравьи, скорее всего, не будут продолжать движение наугад, а будут двигаться по оставленному следу. Феромонные следы испаряются со временем. Длинные маршруты муравьи проходят дольше, а значит, феромонные следы на них успевают испаряться сильнее чем на более коротких. Таким образом, муравьи все чаще будут отдавать предпочтение более коротким маршрутам, постепенно увеличивая на них плотность феромона.

Генетические алгоритмы относятся к разделу эволюционных алгоритмов и моделируют процесс естественного отбора, в ходе которого с помощью скрещивания и мутаций генерируются новые особи (маршруты), наиболее приспособленные из них отбираются для производства следующего поколения и, таким образом, с каждым новым поколением длина маршрута уменьшается.

Разрабатываемая система имеет клиент-серверную архитектуру. Ядром разрабатываемой системы является Backend приложение, оно будет соответствовать архитектурному стилю REST. Данное приложение взаимодействует напрямую только с базой данных, с остальными сервисами обмен информацией осуществляется с помощью HTTP-запросов. Это позволяет развернуть Backend и frontend приложения на разных серверах.

    В процессе разработки используются следующие средства:

  • Backend приложение разрабатывается на языка Java с использованием фреймворка Spring для обработки запросов клиента и Hibernate для работы с базой данных
  • Frontend приложение реализуется на языке Javascript с использованием библиотеки ReactJs
  • Мобильное приложение также разрабатывается на Javascript, при этом используется библиотека React Native
  • В качестве системы контроля версий используется git, версии программного продукта публикуются в удаленном репозитории Bitbucket.
  • Для навигации используются картографические данные Яндекс с помощью JavaScript API Яндекс карт
  • В качестве СУБД используется Postgres SQL
